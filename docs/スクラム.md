# スクラム
アジャイル開発の代表的なフレームワーク。<br>
アジャイル開発：開発アプローチ、考え方の総称。ウォーターフォール型に対して、短い期間で設計からテストまでを繰り返して開発していこうとするやり方。
<br>
<br>

* ## スプリント
スクラムにおける工程の反復単位のこと。通常1〜4週間程度の期間に設定され、この期間中にシステムに必要な機能の設計、開発、テストまでを行う。

* ## プロダクトオーナー
製品開発における方向性を決める責任者。プロダクト(製品)の価値を最大限化させることに責任を持つ職務。主にスクラム開発に標準的な職務として導入されている。

* ## スクラムマスター
スクラムフレームワークが確実に遵守され、チームがワークフローを改善できるようサポートするコーチ。セレモニーの進行役、ブロッカーの除去、コラボレーションの促進、チームの自己管理のサポートといった責任を負う。

* ## プロダクトバックログ
開発が必要な機能や改善が必要なものに優先順位をつけたリスト。特にアジャイル開発においてToDoリストのような役割を果たす。

* ## 振り返り
レトロスペクティブのこと。スプリントで、”何がうまくいったのか”、”何がうまくいかなかったのか”を振り返る。
